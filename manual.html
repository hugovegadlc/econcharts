econcharts

Publication-quality economic charts from a short, substance-only YAML file. You write the content — titles, data, per-series chart type, domain annotations. The BBVA house style is already encoded, so every chart comes out consistent.

Spec, not code

A chart is declared, not programmed. You describe what the chart shows in plain YAML; a fixed engine validates it and renders it. There is no plotting code to write, no styling to fiddle with, and nothing to keep in sync between charts — change the data and re-run.

The deal: you supply substance (what the chart is about and the numbers behind it); the theme supplies form (colours, fonts, spacing, label formats). If a formatting choice ever shows up in your spec, it belongs in the theme instead.

What you do, start to finish

  1. Unzip the econcharts folder anywhere — no installation, no admin rights.
  2. Open examples/gallery.yaml and edit a chart (or copy one and change it).
  3. Put your workbook in examples/ (or point the spec at wherever it lives).
  4. Double-click run.bat.
  5. Collect your figures and a ready PowerPoint deck from examples/_gallery/.

From the command line the same thing is:

econcharts build examples\gallery.yaml

The rest of this manual is the reference for what goes inside that YAML — the chart spec (below), the named style choices you can make (vocabularies), and how to render many charts at once into a deck (batches). Every example in the gallery is a real, runnable spec shown next to the chart it produces.

Quickstart

A chart is one entry in a batch file. Here is a complete, runnable batch with a single chart — copy it into a file called my_charts.yaml:

# my_charts.yaml
output_dir: figuras          # where figures (and the deck) are written
theme: bbva                  # the house style every chart inherits

charts:
  - id: inflacion            # names the output file: inflacion_<date>.png
    title: Inflación — Perú
    subtitle: var. % anual del IPC
    period: 2023Q1:2024Q4    # the time window the x-axis spans
    series:
      - {name: Inflación, type: line, data: [8.7, 7.9, 6.4, 5.1, 3.7, 3.2, 2.8, 2.0]}

Render it:

econcharts build my_charts.yaml

That writes figuras/inflacion_<date>.png — already in BBVA style, with es-PE number and date formatting, a bottom legend, and the right export size:

Reading the spec

output_dirFolder for the figures and the deck (relative to the batch file).
themeThe house style. bbva is the only theme for now; every chart inherits it.
chartsThe list of charts — one figure each.
idA unique name per chart; it becomes the output filename.
title, subtitleText above the chart. Both optional.
periodThe window the x-axis spans (see Period).
seriesThe lines/bars/areas. Each needs a name, a type, and data.
To add a second chart, copy the whole - id: … block, give it a new id, and edit it. To plot real numbers, point data at a workbook instead of an inline list — see Data.

The chart spec

A chart is a mapping of keys. Only series is required, and within it every series must state its type — there is no chart-wide default. Everything else is optional and falls back to the theme.

titleChart title. Optional.
subtitleSmaller line under the title. Optional.
sourceKept as metadata but not drawn — it belongs on the slide beside the chart.
periodThe window the x-axis spans (see Period).
themeHouse style. Default bbva.
ylabel, y2labelAxis labels (left / right). Optional.
date_labelPick a date-label style by name (see vocabularies).
seriesRequired. One or more series — see below.
annotationsReference lines, bands, shaded periods (see Annotations).
styleEscape hatch for raw matplotlib overrides. Rarely needed.

Series

Each entry in series is one plotted line, bar, or area.

nameRequired. Identity + default legend text.
typeRequired. line · bar · area · stacked.
dataRequired. A workbook reference or inline values (see Data).
axisprimary (default) or secondary (right-hand scale).
labelOverrides the legend text (default = name).
colorPin a palette colour by name (see vocabularies).
linesolid (default) · dashed · dotted. Line series only.
markData labels / dots (see Marks).

The type is also the combination rule. Several bar series sit side by side (dodged); several stacked series stack (negatives downward); area series fill and stack; line series are smoothed curves drawn on top. You can mix them — the classic combo is stacked contributions with a total line:

series:
  - {name: Consumo,   type: stacked, data: "excel:pbi.xlsx#trim!consumo"}
  - {name: Inversión, type: stacked, data: "excel:pbi.xlsx#trim!inversion"}
  - {name: PBI,       type: line,    data: "excel:pbi.xlsx#trim!pbi"}

Marks

A mark labels a series' values at chosen points.

atWhich points: all, last, a period token, or a list of tokens.
valueShow the numeric value as a label. Default true.
markerDraw a dot at the point. Line series only. Default false.
textCustom text replacing the value (single point only).
series:
  - {name: PBI, type: line, data: "…", mark: all}                        # label every point
  - {name: IPC, type: line, data: "…", mark: {at: last, marker: true}}   # dot + value at the end

Shorthand: mark: last is the same as mark: {at: last}. last means that series' last real point — which can differ from the end of the sample.

Annotations

Non-data markup, listed under annotations. Each is one tagged map.

hlineHorizontal reference line(s): hline: 0 or [0, 2].
vlineVertical line(s) at period tokens: vline: 2020Q2.
spanShaded period: {from, to, label?}.
bandShaded horizontal range: {y0, y1, label?}.

Lines take color, weight (thin/thick) and line (solid/dotted); fills take color. These use a small annotation colour vocabulary — grey · orange · blue.

annotations:
  - {hline: 0}
  - {vline: 2023Q1, color: orange, weight: thick, line: dotted}
  - {span: {from: 2022Q4, to: 2023Q3, label: Contracción}, color: orange}
  - {band: {y0: 1, y1: 3, label: Meta BCRP}}

Data

A series' data takes one of three forms:

"excel:file#sheet!column"A workbook column. The file resolves next to the batch; the sheet's first column holds the dates.
[1, 2, 3, …]Inline values, lined up to the period window.
{2024Q1: 1.2, …}Inline {period: value} pairs (they carry their own dates).

Missing values (null) may sit only at the ends of a series; the line starts and ends at its first and last real point (no bridging) — which is exactly how a forecast overlay picks up where the observed line stops.

Period

period is the axis frame: the chart spans exactly that window, even where a series has no data; anything outside it is clipped. With no period, the axis fits the data. It is a single token, or start:end (inclusive):

Year2024
Quarter2024Q1
Month2024M03
Day (ISO)2024-03-15

Either bound can be a data-driven token: start = the sample's first period, end = its last (across all series). So 2018Q1:end always reaches the latest data, and start:end spans exactly what you have. One frequency per chart; the axis decides how to label it (daily data over a long span is labelled by month, or year).

Style vocabularies

You never write a colour or a font size. When you want to override the theme's automatic choice, you pick a name the theme already defined — the same way color: orange picks a palette colour. The theme owns the actual values; you choose among sanctioned names. The only place raw values are allowed is the style: escape hatch.

Line style · line

solidDefault.
dashedReads as a forecast / scenario / trend line.
dottedA lighter alternative.

Date labels · date_label

The axis chooses the granularity to fit the width — daily data collapses to month, quarter, or year as the span grows. Each granularity has named styles; date_label: <name> selects one where it applies:

Dayplain → 15-jul  ·  dotted → 15-jul.
Monthplain → mar-24  ·  dotted → mar.-24
Quartermonth → mar-24 (default)  ·  quarter → 1T24
Yearfull → 2024

Export size

A render-time choice — one spec can target several — set on the batch header or a chart, not inside the chart body. Physical millimetres:

slides_half85 × 70 mm — the default.
slides_full140 × 75 mm
word_half75 × 60 mm
word_full117 × 60 mm
The cascade. A named choice resolves most-general to most-specific: theme default → batch header → individual chart. So a deck can default to one quarter-label style and a single chart override it.

Batches & decks

A batch file renders many charts in one run. It has a small header — where data and outputs live, plus the defaults every chart inherits — and a list of charts:

output_dir: figuras
theme: bbva
size: slides_half          # defaults every chart inherits

charts:
  - id: inflacion
    title: Inflación — Perú
    series: [ … ]
  - id: pbi
    size: slides_full      # this chart overrides the default
    series: [ … ]

Header keys

data_rootWhere workbooks live (default: next to the batch file).
output_dirWhere figures and the deck are written (default figuras).
theme, size, backend, date_labelDefaults every chart inherits; a chart may override any.
renderOptional list of ids to render (omit = all).

Paths resolve relative to the batch file, so the whole folder can be moved or shared and still work.

Output names

Each chart becomes <id>_<date>.png — the id you gave it plus the render date, e.g. inflacion_20260611.png. Re-running on the same day refreshes them; it asks once before overwriting.

Running

econcharts build batch.yaml                       # all charts
econcharts build batch.yaml --only pbi,inflacion  # a subset
econcharts render one_chart.yaml -o chart.png     # a single chart

If a chart has an error it is reported by id and the rest still render — one bad chart never sinks the batch.
